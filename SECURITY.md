# Security

This repo is designed to be forked and adapted. Treat any company-specific fork as private if it contains customer, prospect, employee, investor, merchant, borrower, payment, or internal operating data.

## Do not commit

- API keys or access tokens.
- CRM, payment, banking, borrower, customer, or prospect exports from a real company.
- Private board, investor, employee, or financial data.
- Internal URLs, credentials, or system identifiers.

## Recommended use

Use the included sample data first. If you adapt this for a real company, keep the fork private and store secrets in environment variables or your company's secret manager.
